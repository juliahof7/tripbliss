# TripBliss ✈️

    TripBliss is a mobile-first travel planning web application designed to help users organize their trips in one place. Built as a capstone project for a Bachelor of Computer Science degree with a focus on User Experience (UX) and Software Development.

## 🌍 Live Demo
[https://tripbliss-indol.vercel.app/](https://tripbliss-indol.vercel.app/)

## 📱 About the Project
    Many travelers save destinations, hotels, restaurants, and activities across multiple platforms such as notes apps, browser bookmarks, and social media. TripBliss solves this problem by providing a centralized space where users can archive past trips while also planning future adventures through organized boards.

## ✨ Features
- **Splash Screen** — Branded intro screen with auto-advance
- **Onboarding Flow** — 4-screen onboarding experience for new users
- **Sign In / Register** — Account creation and authentication screens
- **Home Dashboard** — Central hub with quick access to all features
- **My Trips** — Pinterest-style trip boards with photo carousels
- **Trip Board Detail** — View all photos, edit notes and trip dates
- **Card Detail** — View detailed info for hotels, restaurants, and activities
- **Browse** — Discover destinations with a searchable photo grid
- **Create a Trip** — Form to create new trip boards
- **Share** — Share trips and cards to social media
- **Profile / Settings** — User profile with notification and theme toggles

## 🛠️ Tech Stack
- **React** — Frontend framework
- **Vite** — Build tool and dev server
- **CSS** — Custom styling with CSS variables
- **Lucide React** — Icon library
- **Vercel** — Deployment and hosting
- **GitHub** — Version control

## 🚀 Running Locally
1. Clone the repository: git clone https://github.com/juliahof7/tripbliss.git
2. Navigate into the project: cd tripbliss
3. Install dependencies: npm install 
4. Start the dev server: npm run dev
5. Open your browser to `http://localhost:5173`

## 🎨 Design
    High-fidelity wireframes were designed in Figma before development began. The app follows a mobile-first design approach with a fixed 390px width to simulate a mobile device experience.
